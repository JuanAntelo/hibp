hibp

A Promise-based API for querying the 'Have I been pwned?' service.

View on GitHub Download .zip Download .tar.gz

Installation

Local via npm:

npm install --save hibp

Browser via CDN (see below for more information):

<script src="https://unpkg.com/hibp"></script>

Features

  • Get all breaches for an account
  • Get all breached sites in the system
  • Get a single breached site
  • Get all data classes
  • Get all pastes for an account
  • All queries return a Promise
  • Available server-side (Node.js) and client-side (browser)

Usage

import hibp from 'hibp';

Now the following functions are available in the hibp object:

  • .breachedAccount(account, [options])
  • .breaches([options])
  • .breach(breachName)
  • .dataClasses()
  • .pasteAccount(email)
Example:
import hibp from 'hibp';

hibp
  .breachedAccount('rick.sanchez')
  .then((data) => {
    if (data) {
      // Bummer...
      console.log(data);
    } else {
      // Phew! We're clear.
      console.log('Good news â€” no pwnage found!');
    }
  })
  .catch((err) => {
    // Something went wrong.
    console.log(err.message);
  });

Please see the API reference for more detailed usage information and additional examples.

Using in Node.js < 0.12

This module requires a Promise implementation to exist in the global namespace prior to being loaded. Therefore, to facilitate use on versions of Node.js before 0.12, you are responsible for providing a polyfill. I recommend es6-promise.

Example:
// This must be first!
require('es6-promise/auto');

var hibp = require('hibp');

Using in the browser

There is a Universal Module Definition (UMD) build provided in the package dist directory for usage in the browser. When using this build, an hibp object will be added to the browser's window object. Development and production (minified) UMD builds are both provided for download:

  • https://unpkg.com/hibp/dist/hibp.js
  • https://unpkg.com/hibp/dist/hibp.min.js

You can include one of these builds directly via CDN (this example will reference the latest tag version of the production build by default, but you can specify a particular version if desired - see unpkg for details):

<script src="https://unpkg.com/hibp"></script>

Alternatively, you may bundle it in with client-side code with a module bundler like webpack. If your build process honors the browser field in package.json, you can import or require it normally:

// ES2015
import hibp from 'hibp';

// ES5
var hibp = require('hibp');

If your build process does not respect the browser field of package.json, you may explicitly include or require the UMD version like so:

// ES2015
import hibp from 'hibp/dist/hibp.min.js';

// ES5
var hibp = require('hibp/dist/hibp.min.js');

N.B. As stated above, this module requires a Promise implementation to exist in the global namespace prior to being loaded. Therefore, to facilitate usage on browsers without native Promise support, you are responsible for providing a polyfill. Again, I recommend es6-promise.
